Разработать архиватор строк
Вам нужно разработать класс который будет эмулировать архивацию
В нем будет два метода Первый (compress) принимает строку и возвращает байтовый массив
второй (decompress) наоборот.
Compress будет сначала сканировать строку и искать максимальный размер повторяющейся
подстроки.
Например для "ababc" это будет 1, для "ababc" 2/
Потом во время второго сканирования исходная строка превращается в байтовый массив путем
замены повторяющихся строк на некоторые уникальные идентификаторы.
decompress принимает байтовый массив и заменяет идентификаторы обратно на подстроки с
целью получения исходной строки.


Несколько комментариев
1)  Нужно  подправить  ошибки. Например, для строки "aaaa" я получаю в
ответ "aa&0", а для "aaaaa" - исключение.
2) Здесь подразумевалось только работа со строками, поэтому попробуйте
не использовать Java I/O и потоки
3)  Для  исходной строки в 500 символов байтовый массив занимает около
3000  байт,  то  есть  в  три раза больше, чем сама строка. Попробуйте
добиться большего сжатия.
4)  Метод  compress  каждый раз в цикле ищет повторяющую строку, таким
образом  для  50 повторяющихся строк будет 50 проходов. Попробуйте свести это к
одному проходу.
